使用方法一:

xcodebuild
[-project <projectname>] #项目名称
[[-target <targetname>]...|-alltargets] #target名称
[-configuration <configurationname>] #configuration名称
[-arch <architecture>]...
[-sdk [<sdkname>|<sdkpath>]] #SDK名称
[-showBuildSettings]
[<buildsetting>=<value>]... 
[<buildaction>]...

-list

打非发布包时可用下面的方法来提高打包速度：

xcodebuild -quiet -configuration ${build_config} || exit

build_config 可以使Debug Release，会在${build_path}/Debug-iphoneos（或${build_path}/Release-iphoneos）生成一个JellyBlast.app，把JellyBlast.app放入Payload文件夹下，然后直接用zip命令生成ipa：

zip -qr ${project_path}/tmp.ipa Payload

打发布包可用下面的命令：

xcodebuild -quiet -target "${build_target}"  -scheme "${build_scheme}" -configuration ${build_config} archive -archivePath ${archive_path}

会得到一个JellyBlast.xcarchive文件，然后用以下命令生成一个exportOptions.plist

/usr/libexec/PlistBuddy -c "add :teamID string ${tmp_team}" -c "add :method string ${export_plist_method}" -c "add:uploadSymbols bool false" -c "add provisioningProfiles:${bundle_id} string ${tmp_provision_name}" -c "add provisioningProfiles:${bundle_id}.JellyNotificationService string ${tmp_push_provision_name}" -c "add :signingCertificate string iPhone Distribution" -c "add :signingStyle string manual" -c "add:stripSwiftSymbols bool true" ${export_plist_file}

然后把JellyBlast.xcarchive达成ipa包

xcodebuild -exportArchive -exportPath ${ipa_path} -archivePath ${archive_path} -exportOptionsPlist ${export_plist_file}

遇到的问题：

configuration改为Debug后，装包装不上，提示cpu type不匹配。修改所有项目、tartget及静态库的ONLY_ACTIVE_ARCH改成NO，意为debug时编译所有arm类型（原来是Yes）。如果有项目没有统一这个配置则打包会报这个错误：Invalid bitcode signature。

iOS 添加bugly遇到的问题：

有崩溃信息后看不到具体代码行。看上去已经匹配了符号表，但是还是定位不到行，需要修改下面两个配置（最好每个项目都改一下）：

XCode -> Build Settings -> Code Generation -> Generate Debug Symbols -> Yes
XCode -> Build Settings -> Build Option -> Debug Information Format -> DWARF with dSYM File

这样重新打包后，dsym的uuid仍然没有变，但是可以定位到代码行了。（后来发现又出现定位不了的情况，发现Generate Debug Symbols -> Yes在项目文件中被删掉了，但是xcode中还是显示有，是因为这个值默认是yes，打包的时候缺失导致定位不到代码行）
